★ Priority 1 · RELEASE GATE

Deauth frames on-air (platform-fork validation)

The whole v0.4 migration exists to answer one question. Stock IDF 5.3 blob filtered MGMT subtypes 0xC / 0xA at the TX FIFO. Bruce's patched libnet80211.a lifts the filter. This test confirms that actually happened.

  • Flash master, boot, confirm splash reads v0.4
  • Pick a non-PMF WPA2 AP you own; associate a phone / laptop to it
  • POSEIDON → WiFi → Scan → select target → press D
  • Watch the counters: sent: climbs hundreds/sec, drops: stays near zero
  • Target client drops WiFi within 2-5 seconds
Success signal: reply "deauth verified v0.4" on GitHub Issues. That's the tag-trigger.
Priority 2 · SaltyJack LAN arsenal

RaspyJack port end-to-end

Full homage port of @7h30th3r0n3's RaspyJack. Six new attack modules behind the j menu key. Run against networks you own.

  • DHCP Starve: join target WiFi → run → watch pool exhaust (NAK ≥ 20)
  • Rogue DHCP: STA mode races real server; AP mode is the server
  • Responder: poison LLMNR/NBT-NS, capture NTLMv2 hashes to SD
  • WPAD: serve PAC + HTTP 407 challenge; confirm Windows clients auto-auth
  • NTLMv2 cracker: feed captured hash + wordlist, verify it runs on-device
  • Report: which modules landed, any crashes, pcap/hash-file output paths
Every file in src/features/saltyjack/ is authorized-testing-only. Don't hunt strangers.
Priority 3 · TRIDENT (ESP32-C5 satellite)

End-to-end ESP-NOW with the C5 node

TRIDENT is the ESP32-C5 plug-in satellite that gives POSEIDON 5 GHz deauth, Zigbee sniff, and PMKID capture over ESP-NOW on channel 1. v0.5 is the first release where the whole path works — pairing, scan, attack, and return-to-channel-1 so POSEIDON keeps hearing HELLOs.

  • Flash TRIDENT via the install page web flasher (Chrome/Edge)
  • Power it on — Radio → C5 → Status should show "C5 x1" within a few seconds
  • Run C5 → Scan — expect both 2.4 + 5 GHz APs with auth type column; WPA3 rows show red
  • Run C5 → Deauth → broadcast on a channel you own — watch frames counter tick
  • File any issue where TRIDENT drops off after scan/deauth (should always return to ch 1)
Priority 4 · Regression sweep

Everything v0.3 shipped — still works?

v0.4 migrated the entire platform. Library versions stayed, code stayed — but some corners might've drifted. Touch every menu leaf at least once.

  • Radios: WiFi scan, BLE scan, LoRa scan, CC1101 scan, nRF24 scan — all enumerate
  • Capture: Sub-GHz record+replay, PMKID, 4-way handshake (.22000 format)
  • Meshtastic: Chat, Nodes, Page, Pos — receive real traffic if within range
  • Peripherals: SD browse, BadUSB, IR TV-B-Gone, GPS fix, theme + SFX toggle
  • File any regression with feature path, serial log, reproducibility count

Ideal bug report

  • Build: v0.4 / commit SHA (confirm in boot splash)
  • Hardware: Cardputer-Adv + which hat attached
  • Feature path: e.g. WiFi → Deauth → specific target
  • What happened vs what you expected
  • Reproducibility: always / sometimes / once
  • Serial log if possible (pio device monitor --baud 115200)
