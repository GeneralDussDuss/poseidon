SaltyJack — the LAN arsenal

A direct port & homage to @7h30th3r0n3's RaspyJack and Evil-M5Project, reforged for POSEIDON. Press j from the root menu → custom pirate-themed submenu renderer with list / grid / carousel view toggles, 16×16 RGB565 sprite pack, pixel-doubled for scale, and a 240×135 boot splash baked right into flash.

DHCP Starvation

Floods the target network's DHCP server with random-MAC Discover/Request cycles until the pool exhausts. Live counters for discover/offer/request/ACK/NAK. Auto-detects pool exhaustion on NAK ≥ 20.

PHASE 1

Rogue DHCP (STA + AP)

Races the real server with our own Offer/Ack (STA mode) or serves DHCP itself for clients on our SoftAP (AP mode). Gateway + DNS poisoning for chained MitM.

PHASE 2

Responder

LLMNR + NBT-NS name-poisoning plus SMB1 NTLMv2 Type-2 builder. Captured NTLMv2 hashes log straight to SD in hashcat mode-5600 format. Feed to the on-device cracker.

PHASE 2

WPAD NTLM harvest

Serves a PAC file on port 80 + HTTP 407 Proxy-Authenticate challenge. Windows clients auto-fetch WPAD, auto-auth with current domain creds — hash captured, logged, crack-ready.

PHASE 2

On-device NTLMv2 cracker

Pure-C HMAC-MD5 wordlist runner. Reads hashcat-mode-5600 lines from SD, tries each password. No PC needed. Seeds a starter wordlist on first run.

PHASE 2

Custom UX

Own renderer — 3 view modes (list/grid/carousel), RaspyJack-faithful per-tool info pages, idle screensaver, and a pirate 7-sprite icon pack in 16×16 RGB565 baked from PNG via tools/sprite_sheet_to_icons.py.

UX

Two back-to-back audits landed simultaneously — deauth wasn't kicking clients, LoRa was hard-rebooting the device. Both classes of bug found, fixed, reviewed, shipped.

05 // FIELD TEAM

For testers

v0.4 is live on master and needs eyes before the tag cuts. Priority 1: confirm deauth frames actually land on-air (the platform migration's entire purpose) — flash master, hit WiFi → Deauth on a non-PMF WPA2 AP, watch sent: climb with drops: near zero. Priority 2: the whole SaltyJack arsenal (DHCP starve / rogue DHCP / Responder / WPAD / NTLMv2 cracker) — against networks you own. Full doc: TESTERS.md.

Priority 1 · High Signal

Deauth actually lands now

Primary path addr1 was self-addressed to the AP's BSSID — no client ever saw a frame. Plus static seq=0, no disassoc pair, no unicast follow-up. All fixed.

  • Pick a known WPA2 AP you own
  • Associate a phone / laptop
  • POSEIDON → WiFi → Deauth → pick target
  • Watch drops: and sta:N counters
  • Report: did the client disconnect within a few seconds?
WPA3 / WPA2-Enterprise use PMF and won't kick — that's a design limit, not a bug. UI now warns.
Priority 2 · New Feature

Meshtastic node participation

POSEIDON is a full leaf node on the public LongFast channel now. Receive, send, page, optionally report position.

  • RX: open Mesh Nodes, wait 2 min in a mesh-covered area
  • TX broadcast: Mesh Chat → T → type → Enter; verify another Meshtastic client sees it
  • Paging: Mesh Nodes → Enter on a target → send DM
  • Position: Mesh Pos → T → wait GPS fix → B (send now); check if we appear as a pin on other apps
Leaf-only — we don't forward. No CAD-before-TX yet, so we collide slightly more than a polite node.
Priority 3 · Regression

LoRa stability

Pre-v0.3.0 every LoRa feature crashed on frequency select (bandwidth unit bug) or boot-looped the device (PI4IOE antenna switch LoadProhibited). Spectrum analyzer froze and backtick didn't back out.

  • Enter every LoRa feature — Scan, Beacon, Mesh LF, Analyzer, 4 mesh features
  • Analyzer: all 3 modes render, backtick backs out instantly
  • TAB through bands — radio retunes without crash
  • Report: any Guru Meditation panic (grab pio device monitor log)
Priority 4 · Smoke Test

Everything else still works

Confirming v0.2.0 features haven't regressed: sub-GHz capture/replay/spectrum, BLE scan/spam, nRF24 MouseJack, WiFi scan/PMKID/wardrive, BadUSB, IR, GPS, themes, SD, TRIDENT PC bridge.

  • File as a regression if something worked before and doesn't now
  • Include feature path + what specifically broke

Ideal bug report

  • Build: v0.4 / commit SHA (confirm in boot splash)
  • Hardware: Cardputer-Adv + which hat attached
  • Feature path: e.g. WiFi → Deauth → specific target
  • What happened vs what you expected
  • Reproducibility: always / sometimes / once
  • Serial log if possible (pio device monitor --baud 115200)

File at github.com/GeneralDussDuss/poseidon/issues with label v0.4-tester.
